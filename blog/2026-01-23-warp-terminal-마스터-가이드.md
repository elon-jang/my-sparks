# Warp Terminal 마스터 가이드 (v2025 Mac Edition)

본 가이드는 Rust 기반의 최신 터미널인 Warp를 macOS 환경에서 효율적으로 사용하기 위한 핵심 기능과 사용법을 단계별로 설명합니다. 설치부터 인터페이스, 단축키, AI 기능, 협업 도구, 고급 활용까지 필수적인 내용을 다룹니다.

---

### 1단계: 설치 및 기본 설정

Warp는 macOS 10.14 (Mojave) 이상 환경에서 실행됩니다.

#### **1. 설치 (Installation)**

두 가지 방법으로 설치할 수 있습니다. Homebrew를 이용한 설치를 권장합니다.

* **Homebrew 사용:** 터미널에 아래 명령어를 입력합니다.
```bash
brew install --cask warp
```

* **직접 다운로드:** Warp 공식 웹사이트에서 `.dmg` 파일을 다운로드하여 설치합니다.

#### **2. 초기 설정**

* **로그인:** 기본 기능은 로그인 없이 사용 가능하지만, **AI 보조 기능(Agents 3.0)**과 **Warp Drive(설정 및 지식 공유)** 기능을 활용하려면 가입이 필요합니다.
* **쉘(Shell) 설정:** 설치 시 macOS 기본 쉘인 `zsh`가 자동으로 설정됩니다.

---

### 2단계: 핵심 인터페이스 (Modern Terminal)

Warp는 전통적인 텍스트 기반 터미널과 다른 현대적인 인터페이스를 제공합니다.

#### **1. 유니버설 인풋 (Universal Input)**

* **기능:** 일반 텍스트 에디터처럼 마우스 클릭이나 키보드 방향키로 커서를 자유롭게 이동하며 명령어를 수정할 수 있습니다.
* **스마트 컨텍스트:** 입력창에 `@`를 입력하면 파일, 폴더, Git 상태 등을 선택하여 AI에게 문맥 정보로 제공할 수 있습니다.

#### **2. 블록 (Blocks)**

* **기능:** 입력한 명령어와 실행 결과가 하나의 '블록' 단위로 묶입니다.
* **블록 선택:** `Cmd + ↑` / `Cmd + ↓` 키를 사용하여 블록 간에 이동할 수 있습니다.
* **결과 복사:** 특정 블록을 선택한 후 `Cmd + C`를 누르면, 프롬프트를 제외한 명령어와 실행 결과만 깔끔하게 복사됩니다.

### 3단계: 탐색 및 단축키 활용 (Navigation)

효율적인 작업을 위한 필수 단축키와 화면 관리 기능을 설명합니다.

#### **① 탭 (Tab) 이동**

웹 브라우저와 유사한 단축키를 지원합니다.

* **다음 탭:** `⌃ Control + Tab`
* **이전 탭:** `⌃ Control + ⇧ Shift + Tab`
* **특정 탭 이동:** `⌘ Command + 1` ~ `9`

#### **② 화면 분할 (Split Panes)**

하나의 탭 화면을 여러 영역으로 나눌 수 있습니다.

* **우측 분할:** `⌘ Command + D`
* **하단 분할:** `⌘ Command + ⇧ Shift + D`
* **창 이동:** `⌘ Command + [` (이전 창), `⌘ Command + ]` (다음 창)

#### **③ 런치 설정 (Launch Configurations)**

자주 사용하는 탭과 화면 분할 구성을 저장하고 불러올 수 있습니다.

* **저장:** `Cmd + P`로 커맨드 팔레트를 열고 "Save New Launch Configuration"을 검색하여 현재 구성을 저장합니다.
* **실행:** `⌃ Control + ⌘ Command + L`을 눌러 저장된 구성 목록에서 선택하여 실행합니다.

---

### 4단계: AI 및 에이전트 활용 (Agents 3.0)

AI를 활용하여 명령어를 생성하거나 복잡한 작업을 수행할 수 있습니다.

#### **① AI 명령어 생성 (Generate)**

* **단축키:** `⌃ Control + `` (Backtick)
* **기능:** 원하는 작업을 자연어로 입력하면 실행 가능한 쉘 명령어로 변환해 줍니다. 예) "find files larger than 100MB" → 명령어 자동 생성.

#### **② 에이전트 모드 (Agent Mode)**

* **기능:** AI가 복잡한 요청의 해결 계획(Plan)을 세우고, 필요한 터미널 명령어를 실행하거나 코드를 수정합니다.
* **사용법:** 입력창에 "이 프로젝트의 빌드 에러를 수정해줘"와 같이 자연어로 요청합니다.

#### **③ 서드파티 CLI 에이전트 연동**

* **기능:** Claude Code, OpenAI Codex 등 외부 CLI 에이전트를 감지하면 화면에 유틸리티 바를 표시하여 음성 입력, 파일 탐색 등의 GUI 기능을 지원합니다.

---

### 5단계: Warp Drive (지식 공유 및 협업)

팀과 공유할 수 있는 지식 저장소 기능입니다.

#### **① 워크플로우 (Workflows)**

* **기능:** 자주 쓰는 명령어를 템플릿으로 저장하고, 실행 시 `{{변수}}`에 값을 입력받아 사용할 수 있습니다.
* **단축키:** 설정에서 지정한 단축키(예: `⇧ Shift + ⌥ Option + R`)로 호출합니다.

#### **② 노트북 (Notebooks)**

* **기능:** 마크다운 문서 내에 실행 가능한 코드 블록을 포함할 수 있어, 온보딩 문서나 런북(Runbook) 작성에 유용합니다.

#### **③ 환경 변수 및 웹 뷰**

* **환경 변수:** 프로젝트별 환경 변수(`.env`)를 Warp Drive에 저장하여 워크플로우 실행 시 안전하게 사용할 수 있습니다.
* **Warp Drive on Web:** 웹 브라우저에서도 Warp Drive의 내용을 확인하고 링크로 공유할 수 있습니다.

---

### 6단계: 고급 활용 (TMUX & SSH)

#### **① TMUX 사용 가이드**

* **로컬 환경:** Warp는 자체적인 화면 분할과 런치 설정 기능을 제공하므로, 로컬에서는 `tmux`를 대체하여 사용하는 것을 권장합니다.
* **원격 환경 (SSH):** SSH 접속 시 원격 서버에서 **Warpify** 기능을 사용하면 원격 터미널에서도 Warp의 블록 및 AI 기능을 사용할 수 있습니다. 접속 후 `Cmd + P` > "Warpify Subshell"을 실행하세요.
